move(req);

The function move is called every single time your bot is expected to playa a move. This is the best function because it's where all the fun happens. You write all of the logic that makes your bot work in this function. When it's your turn this function will be called. There are two main parts to this function.
Part 1: Randomly decide on row to play. Math.random() choose a number between 0 and 1 (0 ≤ num < 1) scales by the amout of rows in a board, and then rounds down.
Part 2: Chcek if the move you want to play is avalible. Checks if the move is legal than exits the forever loop if it is. 

async function move(req){
    // console.log("got move request");
    const json = await req.json();
    const board = json.board;
    let row;
    while(true){

	//---- part 1 ----
        row = Math.floor(Math.random() * rows);

	//---- part 2 ----
        if(play(board,row,1)){
            break;
        }
    }

    return new Response(JSON.stringify({row: row}));

}

Part 2: Specifications

This section defines the specs of the IO with the refree system.

Terms:
Refree - refers to the command and control center that keeps board state, scores, analytics, and reports back to the main API running on the host server.
Bot/Player - The piece of code uploaded by a user to play connect4. The term bot and player will be used interchangebly
Main API -The http server running accepting request, initilzing matches, and controling the database with scores, ratings, wins, losses, ties, and file names.

Where do the Refree and Players run?

The refree and players run inside a docker compose with 3 containers -- 2 players and 1 refree -- each bot is given it's own docker network seprate from eachother. The refree is attached to 3 different networks, 1 for each bot, and a third external network to talk to the Main API.

How do the Refree and Players talk?

The refree and players shall comunicate via the HTTP protocol. Each player shall host a HTTP server on port 3001 with 4 different end points that will return at minimun a 200 OK response. The 4 different end points are defined as follows; /start will be defined as being called once when the Bot is first loaded, and expects no content to be returned. /move will be defined as being called anytime the refree needs a move from this player, and expects a json data containing the varible row. row shall be a integer between 0 and rows - 1 (rows being the maximun number of rows on the board). /reset will be defined as being called anytime the refree clears/resets the board state, and expects no content to be returned. /end will be defined as being called when the refree is about to exit, no content is to be return BUT the player must exit either before or after returning back to the refree. My recomendation is to setTimeout to process.exit(0); on a delay, I chose 100ms.

WOW! That was alot. Basicly run a HTTP server with 4 endpoints.
/start return nothing.
/move return a json with row a row you want to play in. example: { "row" : 3 }
/reset return nothing.
/end exit the process.
